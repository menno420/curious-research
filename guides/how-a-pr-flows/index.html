How a change travels: branch → PR → gate → merge

This is the one loop this whole repo runs on. Watch a single change (the amber dot) make the trip. Nothing here can break main until the very last step.

What you just watched

1. A branch copies the project so you can change anything without risk — main stays untouched the whole time.

2. A Pull Request (PR) is you bringing the piece to the bench: "here's my change, look it over."

3. The gate (a robot checker called substrate-gate) test-fits the change automatically. Red means "not yet" — nothing bad can slip through.

4. Merge bolts the change into main. Claude's PRs merge themselves the moment the gate turns green; PRs you make wait for your click.

Simplification note: real PRs can hold many commits (many dots) and get review comments before merging — the trip is the same.
