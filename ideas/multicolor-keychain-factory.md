# Multicolor name-keychain factory — an OpenSCAD text generator any printer can two-tone; the "3-color" part is the finicky stretch, not the start

> **State:** grown via the ritual (2026-07-13) → verdict: **build** — a parametric OpenSCAD name-keychain generator + a batch gift-plate, starting with a two-color *layer-change* design that even the small single-color printer does with one manual filament swap and no wipe tower. True multi-color-per-layer on the 3-color unit is the labeled stretch (that's where filament waste lives). Build path in "First steps" below.

The captured one-liner: *"A parametric name-keychain generator tuned to the 3-color printer, with a batch plate-layout script for gift runs."* Grown honestly, the tuning point flips: the best *first* version isn't tuned to the 3-color printer at all. It's a stacked two-color design — base plate one color, raised letters a second — where the color swap happens at a single layer boundary, so **any** single-nozzle printer prints it with one manual filament change and zero purge tower. The 3-color, colors-sharing-a-layer version is real and cool, but it's exactly where filament waste lives — a stretch, not the start.

*(Bench terms, one clause each: **OpenSCAD** = a free CAD program where the model is a text script — you write code, it renders a solid — which is the natural fit for "Claude writes the design, you render and export it"; **`text()` / `linear_extrude`** = OpenSCAD's two commands for making flat letters and pulling them up into 3D; **STL** = the mesh file a slicer eats, so "render then export STL" is the handoff out of OpenSCAD; **slicer** = the program that turns a 3D model into the layer-by-layer instructions the printer runs; **layer-based color change / M600** = a command (or the slicer's "Add color change" on the layer slider) that pauses the print at a chosen height so you swap filament by hand; **AMS / MMU / multi-material unit** = the accessory that auto-swaps filaments mid-print — his "3-color" capability — needed only when colors sit side-by-side in the same layer; **wipe tower / purge** = the throwaway pillar the printer builds to flush the old color out of the nozzle before the new one, i.e. the waste that scales with the number of swaps; **flush / purge volume** = how much filament is thrown away per color change, measured in mm³.)*

## The 8 questions

**1. What is this thing, really?**
Two things wearing one sentence. (a) A **parametric generator** — an OpenSCAD script where you set `NAME = "MAX"` plus a couple of sizes at the top and it renders a keychain: a base plate, raised letters, and a ring hole. OpenSCAD's [`text()`](https://en.wikibooks.org/wiki/OpenSCAD_User_Manual/Text) makes the letters as a 2D shape and [`linear_extrude`](https://en.wikibooks.org/wiki/OpenSCAD_User_Manual/Text) pulls them into 3D; the whole keychain is a base `linear_extrude` + extruded `text()` + a `difference()` for the ring hole. This is well-trodden — customizable text-keychain models are abundant ([Thingiverse #2128115](https://www.thingiverse.com/thing:2128115), [Printables #820622](https://www.printables.com/model/820622-customizable-name-keychain-tag), [MakerWorld parametric tag](https://makerworld.com/en/models/1054574-parametric-tag-openscad)). (b) A **batch layout** that tiles a plateful for a gift run. The honest core: the *generator* is genuinely easy and code-native — the "multicolor" and "factory" adjectives are the parts that need honesty, and they're questions 3–5.

**2. What could it grow into? (the fun ceiling)**
A little personal gift press: type a name or a date, get a keychain in your recipient's two colors, and batch a plateful for an event. The ceiling climbs three ways — (a) inlaid multi-color logos or icons using the AMS (colors in the same layer); (b) a family of base shapes from the same script (luggage tags, zipper pulls, bag tags, cable labels); and (c) the crossover — the *same* parametric-text engine stamps labels onto workshop parts, not just gifts ([`drawer-organizer-generator.md`](drawer-organizer-generator.md)). Personalized name/date keychains are a genuinely loved gift category — event favors, anniversary keychains, "people keep them for years" ([Snap Manufacturing — 3D-printed personalized gifts](https://snapmanufacturing.com/top-25-cool-things-to-3d-print-for-a-personalized-gift/)).

**3. What's the coolest version of the simplest build?**
A **two-color stacked keychain, made with one manual swap and no AMS**: the base plate prints in color 1, and at the exact layer where the raised letters begin, you swap once to color 2 and finish. Because the two colors never share a layer, there's **no wipe tower to purge into** — so this works on the small single-color printer too, not just the 3-color one. You add the swap in the slicer by dragging the layer slider to where the letters start and choosing "Add color change," which inserts an [`M600` pause](https://help.prusa3d.com/article/color-change_1687) (Prusa/PrusaSlicer); Bambu Studio supports the same manual change [without AMS hardware](https://wiki.bambulab.com/en/software/bambu-studio/multi-color-printing), and Cura does it via Pause-at-Height. This is precisely the community's documented *ideal first multicolor print* — flat, layer-stacked two-color signs, name tags, and keychains ([Bambu multi-color guide](https://wiki.bambulab.com/en/software/bambu-studio/multi-color-printing), ["flat multicolor prints — coasters, keychains"](https://www.youtube.com/watch?v=jgunDyX0Rbk), [Prusa forum — two-color name tags](https://forum.prusa3d.com/forum/original-prusa-i3-mk3s-mk3-general-discussion-announcements-and-releases/name-tags-in-two-colors/)).

**4. What breaks it? (purge, swaps, batch failure, honest caveats)**
Four honest weak points, and the first is why the "3-color" framing oversells the easy path:
- **True multi-color-per-layer wastes filament — a lot, on small parts.** The moment two colors share a layer (inlaid lettering, a two-tone logo), the printer must swap mid-layer and **purge the nozzle every time**. Purge is a fixed cost per swap regardless of how tiny the part is, so on small multi-color items the thrown-away filament routinely **exceeds the part itself** — this idea's sibling ritual already collected the figures, cross-linked rather than repeated: [`lithophane-night-light.md` §Q4](lithophane-night-light.md) (a small Bambu demo cube: ~83 g purge for an ~11 g part). Adding to that: Bambu Studio's default flush is **~280 mm³ per color change** ([3D Print Mentor](https://3dprintmentor.com/how-to-reduce-purge-waste-in-bambu-studio/), forum-strength), Prusa's MMU documents **~50–150 mm³ per change, directional by color contrast** (black→white costs more than white→black) ([Prusa — purging volumes](https://help.prusa3d.com/article/purging-volumes-mmu_125097), authoritative), and a penguin model went from **9.3 g single-color to 62 g multicolor** — three-quarters of the filament never reaching the part ([3D Print Mentor](https://3dprintmentor.com/how-to-reduce-purge-waste-in-bambu-studio/)).
- **Batching cuts per-item purge but concentrates failure.** Counter-intuitively, printing five *same-color* keychains at once uses [almost the same purge as printing one](https://3dprintmentor.com/how-to-reduce-purge-waste-in-bambu-studio/) — the flush is shared per layer across the plate, so batching *helps* multicolor waste. The catch is the other side of the same coin: a full plate is **one long job, and one knocked-loose part or a clog can ruin the whole plate**. Batch honestly, watch the first layers.
- **"Zero purge" is *nearly* true, not literally.** The layer-change method still oozes a few cm of filament at the resume point ([Prusa color-change doc](https://help.prusa3d.com/article/color-change_1687)) — a few cm of ooze, not an 83 g tower. Don't overclaim.
- **OpenSCAD font reproducibility.** Stick to the fonts OpenSCAD bundles (Liberation Sans/Mono/Serif) so the file renders identically on your machine — exotic or emoji fonts can fail to render ([OpenSCAD text() manual](https://en.wikibooks.org/wiki/OpenSCAD_User_Manual/Text)).

**5. What does it let me build next? (does it compound?)**
Strongly. Three reusable pieces fall out: (a) the **parametric-text engine** — swap the base profile and the same script stamps luggage tags, cable labels, spool-end tags, or drawer faces ([`drawer-organizer-generator.md`](drawer-organizer-generator.md) is the same machine wearing a different base shape); (b) the **one-swap layer-change skill** transfers to *any* two-tone flat print — signs, coasters, bag tags; and (c) the **batch-plate layout** transfers to any "make N of a thing" run. Learning to drive OpenSCAD from parameters is itself the compounding win: it's the "Claude writes the design, you render and export" workflow the whole repo leans on.

**6. What does it need?**
- **Software:** [OpenSCAD](https://en.wikibooks.org/wiki/OpenSCAD_User_Manual/Text) (free) + your normal slicer. Claude writes the `.scad`; you don't even need OpenSCAD's Customizer — Claude edits the parameters at the top of the file directly.
- **Filament:** any two colors you already have (~5–15 g per keychain — a thin flat tag is at the low end).
- **Hardware:** cheap split rings / keyrings (buy a bag). For the auto-swap stretch only: the AMS / 3-color unit.
- **Skills (all beginner):** open a `.scad`, hit **Render (F6)**, **Export STL**; in the slicer, drag the layer slider and choose **Add color change**; swap filament by hand at the pause. Nothing here is advanced.

**7. Who does what — me, Claude, or both?**
- **You:** pick the names, dates, and colors; render + export the STL; slice; add the color change at the letter layer; load filament and swap once at the pause; start and watch the print.
- **Claude:** write the parametric `.scad` (Liberation font, base + raised text + ring hole, all parameters at the top), the batch-plate layout, and a numbered "render → export → add color change at layer N → swap → resume" walkthrough — and, per the repo's teaching doctrine ([`docs/teaching-style.md`](../docs/teaching-style.md)), a short animated HTML explainer showing *why a Z-boundary swap needs no wipe tower but a same-layer swap does*.
- **Both:** decide per keychain — two-color layer-change (the start) vs AMS multi-color-per-layer (the labeled stretch).

**8. What's the smallest piece I could finish this weekend?**
One two-color keychain of one name (placeholder **"MAX"**): Claude hands you the `.scad`, you render and export the STL, slice it, add a single color change at the layer where the letters begin, load an accent color when it pauses, and finish. A **gift-ready object in an hour or two, on either printer, with zero new purchases beyond a keyring**. The following weekend: the batch gift-plate and a couple of colorways.

## Verdict

**build.**

The ritual flipped the one-liner's own premise, and honestly for the better. "Tuned to the 3-color printer" oversells the easy path: the best *first* version is a **two-color layer-change keychain that any single-nozzle printer prints with one manual swap and no purge tower** — which is exactly what the community calls the ideal beginner multicolor print (flat, layer-stacked name tags and keychains). The AMS multi-color-per-layer version is real and genuinely cool, but it's **where the filament waste lives** — purge routinely exceeds the part on small multi-color items (cross-linked to [`lithophane-night-light.md` §Q4](lithophane-night-light.md)), so it's the labeled stretch, not the start. "Factory" is honest at hobby scale — roughly **$0.30 of material for a 15 g keychain** at ~$20/kg PLA ([eufyMake cost breakdown](https://www.eufymake.com/blogs/buying-guides/how-much-do-3d-prints-cost)), and batching *same-color* items shares the purge — as long as it's stated plainly that one long plate risks *one-failure-loses-all* and that hobby cost math ignores your time and machine wear. And the generator itself is a clean, code-native "Claude writes, you render" OpenSCAD build. All of that says build it — just build the honest version first.

**First steps Claude will set up:**
1. **The parametric `.scad`** — base plate + raised text (bundled Liberation Sans) + ring hole, with `NAME`, sizes, plate thickness, and text-layer height as parameters at the top; renders in OpenSCAD, exports a clean STL. Placeholder text only (**"MAX"**, **"WORKSHOP"**), never a real name.
2. **A two-color, one-swap walkthrough** — numbered and every click named: render → export STL → slice → drag the layer slider to where the letters begin → **Add color change** → load the accent color at the pause → resume. Works on the single-color printer with no AMS.
3. **A batch gift-plate layout** — tile N keychains, with the honest note in-line: same-color batching *shares* the purge, but one long plate means one failure can lose the whole run — so watch the first layers.
4. **A short animated HTML explainer** ([`docs/teaching-style.md`](../docs/teaching-style.md) bar) — a Z-boundary swap (no wipe tower) vs a same-layer swap (purge tower), so the multicolor tradeoff is something you can *see*, not just read.
5. **(Stretch, clearly labeled) the AMS multi-color-per-layer version** — inlaid two-tone lettering or a small logo, eyes-open about purge waste and cross-linked to [`lithophane-night-light.md` §Q4](lithophane-night-light.md) so the cost is stated up front, not discovered mid-print.

**Safety — check this yourself:** this is ordinary desktop 3D printing on the standard rails — **Claude designs; you slice and start every print, and you watch it** (repo [`CLAUDE.md`](../CLAUDE.md) §2). No robot-arm motion, no mains wiring, no hot-end mods, nothing load-bearing in this project. The one hands-on moment is the **manual filament swap at the pause**: the nozzle is hot, so handle the *filament*, not the hot end, and let the printer do the parking move. Public-repo rule: keychain text in any committed file stays a **placeholder** ("MAX", "WORKSHOP") — never a real person's name — and don't commit photos of finished gifts that show identifying detail.
